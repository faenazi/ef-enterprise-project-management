# Sprint History

Use this folder to preserve the plan and final status of every implementation sprint.

## Naming Convention

```text
Sprint-01-Plan.md
Sprint-01-Status.md
Sprint-02-Plan.md
Sprint-02-Status.md
```

## Usage Rules

- Create each plan from `../02-Sprint-Plan.md`.
- Create each status record from `../03-Sprint-Status.md`.
- Keep only the current factual position in the active Sprint Status template.
- At sprint closure, save the final status here before starting the next sprint.
- Do not rewrite completed sprint history; record corrections as a new revision.
- Return incomplete scope to the Delivery Backlog or identify its target future sprint.
- Preserve implementation, test, defect, decision, and evidence references.
